## 架构总览

采用"管理 → 执行 → 验收"三Agent架构，通过Codex跨会话消息机制实现自主协作。

```
用户需求 → 管理Agent拆解 → 产品规划Agent规划 →
产品开发Agent实现 → 测试验收Agent审查 → 管理Agent归档
                              ↓
               （审查不通过则返回开发Agent修改）
```

## 协作规则

### 跨会话消息协议
- 管理→规划：[任务] 任务描述+要求
- 规划→开发：[规划] 功能规格+任务列表
- 开发→验收：[提交] 实现功能+代码
- 验收→管理：[报告] 测试结果+建议

### 角色分离原则
1. 管理Agent：只决策不执行，只调度不审查
2. 产品规划Agent：只规划不实现
3. 产品开发Agent：只实现不审查（球员）
4. 测试验收Agent：只审查不实现（裁判）

---

## 使用方式

1. **启动系统**：确保所有Agent线程已创建并确认就位
2. **下达任务**：在当前线程描述任务需求
3. **自动流转**：任务在Agent间按规划→开发→验收自动流转
4. **结果汇报**：最终结果回到管理Agent
5. **迭代优化**：可根据验收反馈进行修改迭代

---

## 核心概念（源自视频教程）

1. **身份注册**：每个Agent创建时注册唯一身份标识
2. **跨会话消息**：利用Codex的线程间消息实现通信
3. **工作日志**：每个操作记录日志，方便追溯
4. **并行处理**：不同Agent可在各自线程中并行工作
5. **上下文隔离**：独立线程避免上下文污染
6. **角色分离**：执行与验收角色严格分开
