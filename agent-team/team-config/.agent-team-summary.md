## 系统架构

```
┌─────────────────────────────────────────────────────────┐
│                    🎯 管理Agent                         │
│              (本线程 - 调度·决策·归档)                    │
└────────┬────────────────────────────────────┬────────────┘
         │ [任务] 需求分析                    │ [报告] 验收结果
         ↓                                    ↑

└──────────────────┘    └──────────────────┘    └──────────────────┘
```

## 协作流程（当前演示）

正在进行的演示任务：**自动化日志分析工具（Log Analyzer Tool）**

1. ✅ 管理Agent → 下达任务给产品规划Agent
2. 🔄 产品规划Agent → 正在编写《产品规划文档》
3. ⏳ 产品开发Agent → 等待接收规划文档
4. ⏳ 测试验收Agent → 等待接收验收请求
5. ⏳ 管理Agent → 等待最终汇报

## 核心技术要点

1. **create_thread** - 创建Agent专属线程作为独立工作空间
2. **send_message_to_thread** - 跨线程消息实现Agent间通信
3. **身份注册** - 每个Agent明确知晓自己的角色、职责、协作规则
4. **工作日志** - 每个操作步骤记录日志，便于追踪
5. **角色分离** - 执行Agent与验收Agent严格分离，避免"既当球员又当裁判"

## 使用方法

1. 在左侧Codex边栏可以看到所有Agent线程
2. 在当前线程（管理Agent）下达任务指令
3. 任务按"规划→开发→验收"自动流转
4. 最终结果汇报回管理Agent
5. 可在每个Agent线程中查看详细工作日志
